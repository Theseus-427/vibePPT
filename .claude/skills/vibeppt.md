# vibePPT

将 PPT 设计图转换为可编辑的 SVG 幻灯片。用户用图像生成工具做好设计图，放入 `input/` 文件夹，运行本 skill 后逐张生成 SVG，输出到 `output/` 文件夹。

## 执行流程

### 第一步：扫描输入

1. 列出 `input/` 目录下所有图片文件（png, jpg, jpeg, webp）
2. 按文件名排序
3. 向用户确认：「扫描到 N 张图片，按以下顺序处理：slide-01.png, slide-02.png, ... 是否开始？」

### 第二步：逐张处理

对每一张图片，按以下循环处理：

```
读取图片 → 分析设计 → 生成SVG → 写入output/ → 读取SVG展示预览 → 等待用户反馈 → 修改或进入下一张
```

#### 2.1 读取图片

使用 Read 工具读取 input/ 目录下的当前图片。

#### 2.2 分析设计

仔细分析图片中的 PPT 设计，提取以下信息：

- **配色方案**：背景色、主色、辅色、强调色、文字色。全部精确到 hex 值（如 `#1a1a2e`）。如果图片中的颜色无法精确取 hex，使用最接近的标准色并注明
- **布局结构**：
  - 标题区域的位置和占画布比例
  - 内容区域（文字区、图表区、图片区）的位置和大小
  - 页脚/页码区域
  - 装饰元素的分布
  - 各区域的边距和间距（相对于 960×540 画布）
- **字体层级**：
  - 主标题：字号、粗细、颜色、对齐方式
  - 副标题：字号、粗细、颜色、对齐方式
  - 正文内容：字号、颜色、行高
  - 标签/注释：字号、颜色
  - 所有字体映射为 Arial/Helvetica/sans-serif
- **视觉元素**：
  - 背景处理（纯色/渐变/纹理）
  - 装饰性形状（矩形、圆形、线条、三角形等）的位置、大小、颜色
  - 图标或符号的位置和样式
  - 分割线的位置和样式
  - 如果有数据图表（柱状图、饼图、折线图），记录其类型、数据、位置
- **文字内容**：提取图片中所有可见文字，包括标题、正文、标签、页码等

在生成 SVG 前，先向用户简要汇报设计分析结果（配色 + 布局），然后直接开始生成。

#### 2.3 生成 SVG

根据设计分析结果，生成完整的 SVG 代码。**严格遵循以下规范：**

**基础结构：**
```xml
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 960 540" width="960" height="540">
  <defs>
    <!-- 渐变、阴影等定义 -->
  </defs>
  <!-- 内容 -->
</svg>
```

**设计还原：**
- 尺寸固定为 960×540（16:9 宽屏，PowerPoint 标准）
- 精确使用分析阶段提取的配色 hex 值
- 还原布局结构，各元素位置比例与参考图一致
- 文字内容与原图一致
- 如果原图有数据图表，使用 SVG `<rect>`、`<path>` 等元素手动绘制

**字体规则：**
- 所有 `<text>` 元素使用 `font-family="Arial, Helvetica, sans-serif"`
- 明确设置每个文本元素的 `font-size`、`font-weight`、`fill`
- 使用 `text-anchor` 控制对齐

**元素组织：**
- 使用语义化 `<g>` 标签分组，每个分组加 `id` 属性：
  - `<g id="background">` — 背景层
  - `<g id="header">` — 标题区
  - `<g id="content">` — 内容区
  - `<g id="chart">` — 图表区（如有）
  - `<g id="footer">` — 页脚区
  - `<g id="decoration">` — 装饰元素
- 元素按从背景到前景的顺序排列（z-index 由 DOM 顺序决定）

**PowerPoint 兼容性（关键）：**
- ❌ 禁止使用 `<foreignObject>`
- ❌ 禁止使用 `<script>` 或事件处理器（`onclick` 等）
- ❌ 禁止使用外部资源引用（外部字体、外部图片）
- ❌ 禁止使用 CSS `position`、`display: flex`、`display: grid`（PowerPoint 对 CSS 支持有限）
- ✅ 使用绝对定位（`x`、`y` 属性）
- ✅ CSS 样式写在 `style` 属性或内嵌 `<style>` 标签中
- ✅ 所有元素明确指定坐标和尺寸
- ✅ 渐变使用 `<linearGradient>` 或 `<radialGradient>` 定义在 `<defs>` 中

**视觉质量：**
- 添加适当的细节：背景装饰、分割线、图标等
- 文本添加适当的 `letter-spacing` 提升可读性
- 为卡片/面板添加微妙的阴影或边框
- 圆角矩形使用 `rx`、`ry` 属性
- 色彩对比度确保文字可读

#### 2.4 写入输出

将生成的 SVG 代码写入 `output/slide-NN.svg`（NN 为两位数序号，如 `slide-01.svg`）。

#### 2.5 展示预览

使用 Read 工具读取刚写入的 SVG 文件，向用户展示：
```
第 N 张幻灯片已生成：output/slide-NN.svg

[SVG 预览内容]
```

然后询问：「这张是否满意？（通过 / 修改意见 / 重新生成 / 跳过）」

#### 2.6 处理反馈

- **通过**：标记为完成，继续处理下一张
- **修改意见**（如「把标题字号调大」「背景色改深一点」「左边距太小」等）：根据具体意见修改 SVG，只改相关部分、保留其他内容，修改后重新写入并展示预览
- **重新生成**：重新分析设计并生成全新 SVG，可尝试不同的实现方式
- **跳过**：标记为跳过，保留已生成的 SVG 但标注待处理，继续下一张

### 第三步：汇总报告

所有图片处理完毕后，输出汇总：

```
vibePPT 处理完成
├── 成功：N 张 → output/slide-01.svg ~ output/slide-NN.svg
├── 跳过：M 张
└── 下一步：将 SVG 文件导入 PowerPoint 即可编辑
```

## 注意事项

- input/ 目录为空时，提示用户先放入设计图
- 如果图片质量太低或不是 PPT 设计图，主动告知用户
- SVG 生成失败时，尝试不同的实现方式重试，最多 3 次
- 在修改循环中，保持之前确认过的元素不变，只修改用户指出的部分
- 所有输出使用中文
